Topicos

E o checkpoint que toda entrega deve passar antes de ser considerada done. Sem QA Gate aprovado, nao ha merge.

Cada criterio de aceitacao deve ser verificado. Passe ou falhe. Evidencia de cada validacao (screenshot, log, link).

PR criado, descricao clara, commits atomicos, sem arquivos desnecessarios. O codigo conta uma historia.

Lint sem erros, formatacao correta, tipos corretos, nomenclatura seguindo convencoes. Automatize o que puder.

Coverage minimo atingido, todos os testes passando, casos de erro cobertos. Sem testes, sem confianca.

Inputs validados, autenticacao/autorizacao corretas, sem secrets hardcoded, sem vulnerabilidades conhecidas.

Tempo de resposta dentro dos limites, queries otimizadas, sem memory leaks. Meca, nao ache.

Codigo documentado, API documentada, README atualizado se necessario. Codigo sem doc e codigo temporario.
