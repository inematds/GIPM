üì• Passo 1: Solicitacao do usuario

O pipeline comeca quando uma solicitacao (request) chega ao sistema. Este passo captura tudo que o usuario enviou.

O que capturar no Passo 1

  • ‚Ä¢Payload: Corpo da requisicao (JSON, form data)
  • ‚Ä¢Headers: Authorization, Content-Type, etc.
  • ‚Ä¢Metadados: IP, timestamp, user-agent
  • ‚Ä¢Identificadores: Request ID unico para rastreamento

üîê Passo 2: Validacao de acesso

Antes de processar qualquer coisa, o sistema deve validar quem esta fazendo a solicitacao e se tem permissao.

üîë Autenticacao

  • ‚Ä¢ Validar token JWT
  • ‚Ä¢ Verificar sessao ativa
  • ‚Ä¢ Checar API key
  • ‚Ä¢ Identificar usuario

üõ°Ô∏è Autorizacao

  • ‚Ä¢ Checar permissoes do usuario
  • ‚Ä¢ Validar acesso ao recurso
  • ‚Ä¢ Verificar quotas/limites
  • ‚Ä¢ Aplicar rate limiting

‚ö†Ô∏è Importante

Nunca pule a validacao de acesso. Este passo e a primeira linha de defesa do sistema.

3

üóÇÔ∏è Passo 3: Coleta de contexto

Apos validar o acesso, o sistema coleta todo o contexto necessario para a IA poder responder adequadamente.

Tipos de Contexto

Contexto do Usuario

Preferencias, historico, configuracoes pessoais

Contexto do Dominio

Dados do negocio, regras especificas, documentos

Contexto da Sessao

Conversas anteriores, estado atual

Contexto do Sistema

Configuracoes globais, limites, politicas

4

üîÄ Fluxo de dados entre passos

Cada passo adiciona informacoes ao objeto de contexto que sera passado adiante.

Passo 1 ‚Üí request_data, metadata
Passo 2 ‚Üí + user_info, permissions
Passo 3 ‚Üí + context_data, history

üíª Implementacao em codigo

async def process_entrada(request: Request):
    # Passo 1: Solicitacao
    pipeline_ctx = PipelineContext(
        request_id=generate_id(),
        raw_request=await request.json()
    )

    # Passo 2: Validacao
    user = await validate_token(request.headers)
    if not user:
        raise HTTPException(401, "Unauthorized")
    pipeline_ctx.user = user

    # Passo 3: Contexto
    pipeline_ctx.context = await collect_context(
        user_id=user.id,
        session_id=request.headers.get("x-session-id")
    )

    return pipeline_ctx
