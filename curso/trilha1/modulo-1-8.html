üìñ Glossario: Governanca, Persona, Pipeline

Governanca (Governance)

Conjunto de regras, limites e controles que definem como a IA pode operar dentro do sistema. Inclui politicas, permissoes, limites de custo e auditoria.

Exemplo de uso:

"A governanca do sistema impede que a IA acesse dados de usuarios menores de idade."

üé≠

Persona

Configuracao que define como a IA deve "pensar" e se expressar. Define tom, nivel de detalhe, foco tematico e estilo de comunicacao. E uma politica de pensamento, nao uma mascara de UX.

Exemplo de uso:

"A persona 'Critical Reviewer' faz a IA apontar falhas e inconsistencias em vez de concordar."

üîÑ

Pipeline (Universal Pipeline)

Sequencia padronizada de 9 passos que toda chamada de IA deve seguir: da requisicao inicial ate o retorno final. Garante consistencia e rastreabilidade.

Os 9 passos:

Request ‚Üí Auth ‚Üí Context ‚Üí Normalize ‚Üí Inject ‚Üí AI Call ‚Üí Persist ‚Üí Materialize ‚Üí Return

Two-Phase Rule (Regra das Duas Fases)

Principio que separa a geracao de conteudo em duas etapas distintas: IA gera estrutura (JSON, outlines), Sistema materializa artefatos finais (PDF, video, documento).

Cognitive Governance (Governanca Cognitiva)

Controle sobre como a IA pensa, nao apenas o que ela produz. Inclui personas, preferencias persistentes e injecao automatica de contexto em toda chamada.

Componentes:

  • ‚Ä¢ Personas: Como a IA deve raciocinar
  • ‚Ä¢ Preferencias: Configuracoes persistentes do usuario
  • ‚Ä¢ Injecao: Contexto adicionado automaticamente

Audit Trail (Trilha de Auditoria)

Registro completo de todas as operacoes do sistema: quem fez, quando, o que pediu, o que a IA respondeu, quanto custou. Permite responder "por que isso aconteceu?" a qualquer momento.

O que registrar:

üìñ Glossario: Anti-Lock-In, Reference Implementation

üîÑ

Anti-Lock-In

Principio que garante que nenhum componente e insubstituivel. O sistema deve funcionar mesmo se trocar o modelo de IA, banco de dados, storage ou qualquer outro servico.

Teste do Anti-Lock-In:

"Se o provider X triplicar o preco amanha, em quanto tempo migro?" ‚Üí Se a resposta for "meses", ha lock-in.

üìö

Reference Implementation (Implementacao de Referencia)

Projeto real que demonstra todos os principios GIPM na pratica. Serve para ensino, validacao e auditoria. Inclui documentacao METHOD_MAPPING que liga cada feature ao principio.

Exemplo deste curso:

NotebookLMX (Trilha 4) e a implementacao de referencia que demonstra GIPM com FastAPI + Gemini.

Materializacao (Materialization)

Processo de transformar dados estruturados em artefatos finais. E a segunda fase da Two-Phase Rule: sistema pega JSON e gera PDF, video, documento, imagem, etc.

Exemplos:

üß™ Quiz de Fixacao

Teste seu entendimento dos termos essenciais do GIPM. Tente responder antes de ver as respostas!

1. O que diferencia "Persona" de uma interface de usuario?

Ver resposta

Persona define como a IA pensa (tom, estilo, foco), nao como ela aparece para o usuario. E uma politica de raciocinio, nao uma mascara visual.

2. Na Two-Phase Rule, quem gera o arquivo PDF final: a IA ou o Sistema?

Ver resposta

O Sistema. A IA gera apenas a estrutura (JSON). O Sistema materializa o artefato final (PDF, video, etc.).

3. O que significa um sistema ter "Anti-Lock-In"?

Ver resposta

Significa que nenhum componente e insubstituivel. O sistema pode trocar modelo de IA, banco de dados ou storage sem reescrever a arquitetura.

4. Qual a diferenca entre Audit Trail e Persistencia?

Ver resposta

Persistencia e o principio de armazenar dados. Audit Trail e o registro especifico que permite rastrear decisoes e responder "por que isso aconteceu?".

5. Em qual Depth Level voce aplicaria governanca completa?

Ver resposta

No nivel Institucional. E o nivel para producao com usuarios externos, onde governanca completa e obrigatoria.
